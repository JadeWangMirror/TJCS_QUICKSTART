# CS10102302课程项目说明
  Author: 赵君峤 叶晨
  Date: 2024/2
## 为什么要进行课程项目？
  - 软件工程的知识来源于人类长期的大型团队软件开发实践（1968-）。因此，对软件工程知识的学习和理解要结合工程化的团队软件开发项目实践来进行
  - 本课程项目尽可能的模拟软件项目的整个实施流程，课程项目的结果要以一个可商用的软件产品作为导向，利用合理的项目管理方式和工具保证软件产品的质量（**课程项目评价标准**）
  - 本课程项目的实施过程以敏捷开发为主，同时涉及RUP统一过程模型中的部分流程
## 课程项目的主题
  1. 为什么限定主题
      - 主题聚焦后有利于项目开展过程的指导
      - 同一个主题的项目之间具有可借鉴和可对比性
      - 主题能够对项目的内容和技术起到引导的作用
      - 具体课程项目要求：合法、健康、有益、可行
  2. 主题
      - 中国高校计算机大赛主题 http://www.c4best.cn
      - 信息管理系统
        - 要求：基于现有框架，完成完整的前、后端系统；实现对某一个领域应用的信息管理的支持
        - 示例：ERP管理系统、网上商城、团队任务管理系统、多人共享甘特图、课程管理系统、校园活动管理系统等
      - 智慧城市平台应用系统
        - 要求：构建完整的系统，实现对某一专题信息的收集和管理、基于AI算法的信息处理、检索、分享，可对接第三方系统
        - 示例：垃圾分类管理智能系统、鸟类保护管理智能系统、无人叫车与路径规划系统等、健康管理系统、智慧诊断系统
      - 基于Pipe-Filter 架构或者插件（微内核）架构的可扩展在线信息处理系统
        - 要求：构建一个完整在线系统，实现对某一类信息的可定制在线处理。提供可扩展的系统架构，方便灵活的扩展系统的功能。
        - 示例：图像智能分析系统、点云智能分析系统系统、文本智能分析系统处理系统
## 如何分组？
  - 每一个软件项目组自由组队，人数 5-7 人，需要适度分工（产品经理；需求工程师；软件工程师；测试工程师），而且又能够兼职。个人分工在不同的项目阶段需要能够切换。
  - 每一个软件项目组会在不同的项目阶段分别承担甲方和乙方的角色（[后详](#课程项目如何开展?)）
  - 每一个软件项目组需要有响亮的小组名字和所有人能够遵守的章程，要进行团队成员的有效管理和定量绩效评价（**课程项目个人评价参考**）
## 课程项目如何开展？
  - 每一个课程项目的实施过程分为四个大的阶段：需求工程，软件设计与开发，软件测试，软件交付（[后详](#阶段1-需求工程)）
  - 一个软件项目组（小组A）负责一个项目（项目X）初期的需求工程和后期的软件功能测试，而软件设计与开发由另外一个软件项目组（小组B）负责
  - 软件项目组A和软件项目组B的结合（甲方乙方）在第一阶段需求工程完成后抽签决定
  - 项目实施过程中，软件项目组A和小组B要同时负责一个产品的质量，形成良好的甲乙方合作关系，并在软件交付后共同接受其他小组的评价
  - 项目开发过程中需要每周举行例会，进行2次以上的集中编程，并且在每一个里程碑结束时进行产品展示
  - 使用Gitlab/Github/华为云DevOps以及华为云服务器作为项目的组织和部署平台
### 阶段1-需求工程
  1. 每一个软件项目组提出一个软件项目需求，撰写一段完整的文字描述 
  2. 学习使用UML语言，对项目需求撰写软件需求说明（SRS）
     - 生成用例图
     - 生成关键用例的顺序图或协作图
  3. 项目需求报告汇报
### 阶段2-软件设计与开发
  1. 软件项目组B（乙方）为主和软件项目组A（甲方）部分人员共同构成项目组
  2. 乙方根据甲方提供的软件需求报告进行需求确认和轻量级软件设计
     - 用户故事
     - 非功能需求
     - 整体设计
  3. 3个冲刺
     - 关键类图设计和关键活动图或状态机设计
     - 代码实现
     - 代码走查与单元测试
     - 项目里程碑展示
### 阶段3-软件测试
  1. 由软件项目组A进行系统功能测试，并反馈Bug 
  2. 由软件项目组B在收到Bug报告后限定时间内进行Bug修正
  3. 由软件项目组A完成简要软件测试报告并由软件项目组B确认
     - 发现Bug统计
     - 修复Bug统计
### 阶段4-软件交付汇报
  1. 由软件项目组A和软件项目组B共同对软件项目组A所提出的软件产品及其开发过程进行总结汇报（以软件项目组B为主，因此每组需要参与两次汇报）
## 课程项目如何打分？
### 需求工程
  - 由其他软件开发组对软件项目组A的软件需求说明书成果进行打分
### 里程碑
  - 展示当前里程碑的需求功能点，软件设计和可运行的软件产品
  - 展示小组的会议、讨论和开发过程
  - 由教师从干系人角度对产品进行打分
### 产品总体汇报
  - 其他软件开发组以及教师从干系人角度对产品进行打分（对组A和组B分别评分，依据组A的需求合理性以及需求变更的响应，以及组B的设计与开发过程和质量）
## 评分计算方法
  1. 每个软件项目组评分 = 所提出软件项目需求工程分（阶段1 + 阶段4）* 0.3 + 所提出软件项目发现Bug加分 * 0.1（阶段3） + 所负责开发软件项目分（阶段2 + 阶段4）* 0.7 - 所负责开发软件项目未及时修复Bug扣分 *0.1（阶段3）
  2. 每个同学项目评分 = 所在软件项目组分 * 个人绩效比例（0~1，组长根据设计任务，功能开发，测试任务，Bug，管理，汇报给出）
  3. 每个未修复Bug扣分: 严重（无法运行，无法部署，软件崩溃） 10分，较为严重（功能缺失，结果错误）5分，一般（偶发错误 出现概率<10%, 界面错误）2分
  4. 每个发现Bug加分：严重 5分，较为严重 2分，一般 1分
