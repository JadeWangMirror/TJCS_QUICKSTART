# Flask 前后端分离后端实验设计

## 实验目标

设计并实现一个基于 Flask 的后端服务，完全与前端分离，提供以下功能：

1. 通过 GET 方法读取 txt 文件内容
2. 通过 POST 方法写入内容到 txt 文件
3. 使用 Postman 进行接口测试

## 实验步骤

### 1. 项目结构

```
flask_backend/
├── app.py            # 主应用文件
├── data/             # 数据存储目录
│   └── notes.txt     # 存储数据的文本文件
└── requirements.txt  # 依赖文件
```

### 2. 实现代码 (app.py)

```python
from flask import Flask, request, jsonify
import os

app = Flask(__name__)

# 确保data目录存在
os.makedirs('data', exist_ok=True)
FILE_PATH = 'data/notes.txt'

@app.route('/api/notes', methods=['GET'])
def get_notes():
    """读取txt文件内容"""
    try:
        with open(FILE_PATH, 'r', encoding='utf-8') as f:
            content = f.read()
        return jsonify({
            'status': 'success',
            'data': content,
            'message': '内容读取成功'
        })
    except FileNotFoundError:
        return jsonify({
            'status': 'success',
            'data': '',
            'message': '文件不存在，已自动创建空文件'
        }), 200
    except Exception as e:
        return jsonify({
            'status': 'error',
            'message': f'读取文件失败: {str(e)}'
        }), 500

@app.route('/api/notes', methods=['POST'])
def add_note():
    """写入内容到txt文件"""
    try:
        data = request.get_json()
        if not data or 'content' not in data:
            return jsonify({
                'status': 'error',
                'message': '缺少content参数'
            }), 400

        content = data['content']

        # 以追加模式写入文件
        with open(FILE_PATH, 'a', encoding='utf-8') as f:
            f.write(content + '\n')

        return jsonify({
            'status': 'success',
            'message': '内容写入成功'
        })
    except Exception as e:
        return jsonify({
            'status': 'error',
            'message': f'写入文件失败: {str(e)}'
        }), 500

if __name__ == '__main__':
    app.run(debug=True, host='0.0.0.0', port=5000)
```

### 3. 依赖文件 (requirements.txt)

```
flask==2.0.1
```

### 4. 安装依赖

```bash
pip install -r requirements.txt
```

### 5. 运行应用

```bash
python app.py
```

## Postman 测试方案

### 1. GET 请求测试

1. 打开 Postman
2. 选择 GET 方法
3. 输入 URL: `http://localhost:5000/api/notes`
4. 点击 "Send"
5. 检查响应:
   - 第一次请求可能返回空内容或文件不存在的提示
   - 后续请求应返回文件内容

### 2. POST 请求测试

1. 打开 Postman
2. 选择 POST 方法
3. 输入 URL: `http://localhost:5000/api/notes`
4. 选择 "Body" > "raw" > "JSON"
5. 输入 JSON 内容:
   
   ```json
   {
    "content": "这是第一条测试笔记"
   }
   ```
6. 点击 "Send"
7. 检查响应:
   - 应返回成功消息
   - 可以再次使用 GET 请求验证内容已写入

### 3. 错误测试

1. 测试不带 content 的 POST 请求
2. 测试 GET 请求时手动删除 data 目录，观察应用是否自动创建
