# TJCS QUICKSTART 整理协作规范

本仓库用于维护同济大学计算机科学与技术专业课程资料。整理目标是让每门课程目录具备统一入口、清晰分类和可持续补充的结构。

## 基本原则

- 保持顶层结构为 `学期/课程名/`，不要把课程资料跨学期或跨课程混放。
- 每门课程目录下放一个**课程级说明文件** `<课程名>.md`，作为该课程的唯一入口；多个学期、多位任课教师共用同一个课程级说明文件。
- 课程目录下还放一个 `reference.md`（参考作业链接，多老师共用）；课程目录下按「学期编码-教师缩写」建子目录（同课不同学期或不同教师则各建一个），存放该学期该教师的教学资料（课件、作业题目、笔记等）。
- 课程目录内零散的说明性 Markdown 应合并进 `<课程名>.md`；合并后如无独立保留价值，应删除原说明文件。
- 移动文件时尽量保留原文件名中的语义信息，不为了统一而抹掉年份、教师、章节、实验编号、组别等关键信息。
- 不确定资料类型或归属时，先保留在原位置，并在 `<课程名>.md` 的对应章节写 `🚧 待补充` 或在「留言区」中说明，不要武断移动。
- 大文件、压缩包、可执行文件和工程产物移动前应谨慎判断，避免破坏项目可运行性。
- 教材不入仓（版权 + 体积），只在 `<课程名>.md` 的「教材」章节写书名/作者，读者自行获取；**例外**：老师自编或公开渠道搜不到的内部教材可入仓，放入对应老师的 `textbook/` 目录。
- 学生提交的作业成果（平时作业、大作业、实验、展示等）不直接入仓，统一以外部链接形式维护在课程级 `reference.md` 中。

## 推荐课程目录结构

每门课程整理后优先使用以下结构。没有对应资料时可以不创建空目录。

```text
学期/课程名/
├── 课程名.md              # 课程级说明，多学期/多教师共用
├── reference.md          # 参考作业链接（多老师共用）
└── 学期编码-教师缩写/       # 如 25262-xxx（学期编码-教师拼音首字母）
    ├── ppt/              # 课件、课堂 PPT
    ├── notes/            # 笔记、复习资料、知识点整理
    ├── exam/             # 期末、期中、考纲、试卷、答案
    ├── textbook/         # 仅老师自编/外部搜不到的内部教材（公开教材不入仓）
    ├── tasks/            # 老师布置的作业内容（题目、要求、指导书）
    │   ├── homework/     # 平时作业（题目、要求）
    │   ├── project/      # 大作业、课程设计（题目、要求）
    │   ├── lab/          # 实验作业（指导、要求）
    │   └── presentation/ # 展示、答辩（要求、模板）
    ├── image/            # 图片、截图、图表
    └── archive/          # zip/rar/7z 等归档包或暂不拆分的历史材料
```

教材不入仓（只在 `<课程名>.md` 写书名）；教学资料（课件、笔记、试卷、作业题目）放在各老师子目录下按类型分目录；学生提交的作业成果只在课程级 `reference.md` 维护外部链接。本仓库只保留课程说明、老师布置的题目与课件、项目报告/展示文档等非源码资料。

## `<课程名>.md` 内容规范

课程级说明文件多学期、多教师共用，建议按以下顺序和格式组织。标题层级保持扁平：以 `## 课程名` 开头，章节用 `###`，作业等子分类用加粗 `**小标题**` 而非更低级标题。没有信息的章节可以省略，但不要写空标题；没有现成内容的章节正文用 `🚧 待补充` 占位。

```markdown
## 课程名

### 📚 教材

### ✏️ 作业

### 🛠 课程工具 / 软件 / 环境

### 📖 期末考试与复习

### ✅ 考勤与小测

### 📊 分数构成与给分

### 🔗 外部学习资料

### 💬 留言区
```

各章节的写法约定：

- **教材**：只写书名（+ 作者/版次），教材文件不入仓。教材通常全课程通用，不分老师。

  ```markdown
  ### 📚 教材
  - 《数据结构（C语言版）》严蔚敏
  ```
- **多老师区分**：作业、考勤、给分等因老师不同的信息，用 `**教师缩写**` 加粗作子标题分块，未整理的老师写 `🚧 待补充`：

  ```markdown
  **平时作业**
  - **wzh**：canvas 发布，手写拍照上传
  - **dzj**：🚧 待补充
  ```
- **资料路径**：课件、笔记、试卷等目录型资料引用时，相对路径带 `学期编码-教师缩写/` 前缀。
- **占位**：没有现成内容的章节或某位老师未整理的部分，统一用 `🚧 待补充` 标记。

外部仓库链接、课程体验、考试范围、给分情况、考勤方式、署名（用 `— *昵称*`）等都应保留。

## `reference.md` 写法

课程目录下放一个 `reference.md`（与 `<课程名>.md` 同级，多老师共用），集中维护该课程的学生作业参考链接（不存放作业实体文件）。

- 按作业类型分节：`## 平时作业` / `## 大作业` / `## 实验` / `## 展示`，与 `tasks/` 下的分类对应；多位老师的作业可在条目里注明老师。
- 每条链接注明贡献者昵称与作业信息，链接指向外部仓库或文件托管，例如：

  ```markdown
  ## 大作业
  - 燕还故榻：[Tiger 编译器实现](https://github.com/xxx/xxx) — 词法/语法分析器 + 中间代码生成
  ```

- 没有内容时整节约略或写 `🚧 待补充`。

## 文件归类规则

以下路径均位于 `学期/课程名/学期编码-教师缩写/` 子目录下（`reference.md` 除外，它在课程目录下）。

- 课件、课堂 PPT：放入 `ppt/`。
- 笔记、复习资料、知识点总结：放入 `notes/`。
- 试卷、答案、考纲、期中期末相关材料：放入 `exam/`。
- 老师布置的作业内容（题目、要求、指导书）：放入 `tasks/` 下对应子目录——平时作业 `tasks/homework/`、大作业与课程设计 `tasks/project/`、实验作业 `tasks/lab/`、展示与答辩 `tasks/presentation/`。
- 图片、图表、截图：放入 `image/`。
- 压缩包：放入 `archive/`，除非压缩包本身是课程提交物或项目交付物的一部分，此时放入对应项目目录。
- 教材文件**不入仓**，书名写在 `<课程名>.md` 的「教材」章节；**例外**：老师自编或公开渠道搜不到的内部教材可放入 `学期-教师/textbook/`。
- 学生提交的作业成果（解答、源码、报告、展示材料等）：**不直接入仓**，以外部链接形式写入课程级 `reference.md`。

## 命名规则

- 课程级说明文件命名为 `<课程名>.md`，放在课程目录下（如 `编译原理/编译原理.md`）。
- 课程下各资料子目录命名为 `学期编码-教师缩写`，如 `25262-wzh`。
  - 学期编码：与学校学期编码一致（如 `25262`）；同课不同学期分别建目录。
  - 教师缩写：任课教师姓名的拼音首字母（小写），如 `wzh`、`dzj`。
- 其他分类目录名优先使用英文小写，如 `ppt`、`notes`、`exam`、`tasks`、`image`、`archive`。
- 课程名、资料文件名可以保留中文，避免无意义重命名。
- 同类材料按课程已有语义命名；如需补充命名，可使用 `年份-类型-说明` 或 `实验编号-说明`。
- `reference.md` 中每条链接注明贡献者昵称（如 `燕还故榻`）和作业信息；昵称/学号只出现在链接描述里，不再作为仓库子目录。
- 文件名避免使用全角括号（如 `（）`）等易被 Windows 文件系统改名的字符，以免反复出现文件名异常；确需标注卷种、序号时用半角 `-` 连接，如 `2021-2022-B卷.docx`。
- 不使用 `新建文件夹`、`资料`、`杂项` 等模糊目录名；确需临时存放时使用 `archive/` 并在说明文件中解释。

## 整理流程

1. 进入 `学期/课程名/` 目录，确认或创建课程级 `<课程名>.md` 和 `reference.md`。
2. 按学期和任课教师创建 `学期编码-教师缩写/` 子目录，把该老师的教学资料归入其下。
3. 读取已有 Markdown、txt 或可读说明文件，确认需要合并到 `<课程名>.md` 的内容。
4. 起草或更新 `<课程名>.md`：教材章节写书名，引用老师目录资料时路径带 `学期-教师/` 前缀。
5. 按文件类型在老师目录下创建必要分类目录（`ppt`/`notes`/`exam`/`tasks/*`）；教材不入仓。
6. 学生作业成果整理成外部链接，写入课程级 `reference.md`。
7. 移动资料文件，并同步更新 `<课程名>.md` 中的相对路径。
8. 检查是否存在空目录、失效链接、重复说明文件和明显误归类。
9. 使用 `git status` 查看改动范围，确认只改动当前课程相关文件。

## 注意事项

- 不要删除原始课程资料，除非它只是已合并的说明文件或明显重复文件。
- 已入仓但不符合「学生作业成果不直接入仓」规范的作业文件/源码，应迁移为外部链接后从本仓库删除；暂时没有链接时，在 `reference.md` 或说明文件中以 `🚧 待补充` 标记。
- 公开教材文件不入仓（书名写 md）；**例外**：老师自编/搜不到的内部教材可入仓 `textbook/`。已在仓内的公开教材，迁移为书名后删除实体。
- 不要修改二进制课程资料内容，只做移动和命名整理。
- 对 Git LFS 管理的大文件保持谨慎，移动后需要确认路径变化是否需要同步更新根 README 或 `.gitattributes`。
- 对包含构建产物的项目，优先记录现状；是否清理 `.o`、`.exe`、`node_modules` 等产物应单独决策。
- 课程级说明文件多老师共用，整理某位老师的资料后，注意说明文件里是否需要补充或区分该老师的信息。
