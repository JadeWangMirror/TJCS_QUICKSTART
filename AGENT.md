# TJCS QUICKSTART 整理协作规范

本仓库用于维护同济大学计算机科学与技术专业课程资料。整理目标是让每门课程目录具备统一入口、清晰分类和可持续补充的结构。

## 基本原则

- 保持顶层结构为 `学期/课程名/`，不要把课程资料跨学期或跨课程混放。
- 每门课程下按「学期编码-教师缩写」建子目录（同课不同学期或不同教师则各建一个）；课程说明文件与目录同名，即 `<学期编码-教师缩写>.md`，作为该学期、该教师开设课程的唯一说明入口。
- 说明文件所在目录内零散的说明性 Markdown 应合并进该 `<学期-教师>.md`；合并后如无独立保留价值，应删除原说明文件。
- 移动文件时尽量保留原文件名中的语义信息，不为了统一而抹掉年份、教师、章节、实验编号、组别等关键信息。
- 不确定资料类型或归属时，先保留在原位置，并在 `<学期-教师>.md` 的对应章节写 `TODO` 或在「留言区」中说明，不要武断移动。
- 大文件、压缩包、可执行文件和工程产物移动前应谨慎判断，避免破坏项目可运行性。
- 课程设计、大作业、完整项目等项目源代码不直接提交到本仓库；应在 `<学期-教师>.md` 的「大作业」章节或 `assignment/project/README.md` 中以外部仓库链接、发布页链接或说明链接的形式维护。

## 推荐课程目录结构

每门课程整理后优先使用以下结构。没有对应资料时可以不创建空目录。

```text
学期/课程名/
└── 学期编码-教师缩写/          # 如 25262-wzh（学期编码-教师拼音首字母）
    ├── 学期编码-教师缩写.md     # 课程说明，文件名必须与目录名完全一致
    ├── doc/
    │   ├── textbook/      # 教材、参考书、讲义
    │   ├── ppt/           # 课件、课堂 PPT
    │   ├── notes/         # 笔记、复习资料、知识点整理
    │   ├── exam/          # 期末、期中、考纲、试卷、答案
    │   └── tasks/         # 老师布置的作业内容（题目、要求、指导书）
    │       ├── lab/       # 实验作业（指导、要求）
    │       ├── project/   # 大作业、课程设计（题目、要求）
    │       └── homework/  # 平时作业（题目、要求）
    ├── assignment/        # 学生提交的作业
    │   ├── homework/      # 平时作业，按 昵称/ 子目录组织
    │   ├── project/       # 大作业，仅维护一个 README.md 链接到自己的仓库
    │   └── lab/           # 实验解答，按 昵称/ 子目录组织
    ├── image/             # 图片、截图、图表
    └── archive/           # zip/rar/7z 等归档包或暂不拆分的历史材料
```

如果课程资料涉及可运行项目，项目源代码应放在独立仓库，并在 `<学期-教师>.md` 的「大作业」章节或 `assignment/project/README.md` 中提供链接。本仓库只保留课程说明、项目报告、项目展示文档、演示视频和必要的非源码资料。

## `<学期-教师>.md` 内容规范

课程说明文件建议按以下顺序组织。没有信息的章节可以省略，但不要写空标题；没有现成内容的章节标题保留，正文留空或写占位说明。

```markdown
# 课程名

## 资料结构

### 教材/参考书/讲义
### 课件/PPT
### 笔记/复习资料/知识点整理
### 作业
#### 平时作业
#### 大作业
#### 实验

## 课程工具/软件/环境

## 期末考试与复习

## 考勤与小测情况

## 分数构成和给分情况

## 外部学习资料

## 留言区
```

整理时应优先把原有说明文件中的有效内容完整迁移到这些章节中，避免只留下文件列表。教材、参考书用 Markdown 超链接给出相对路径；课件、笔记等大量文件可用「见 `doc/ppt/`」的形式指明目录。外部仓库链接、课程体验、考试范围、给分情况、考勤方式等都应保留。

## 文件归类规则

以下路径均相对于 `学期/课程名/学期-教师/`。

- 教材、讲义、参考书：放入 `doc/textbook/`。
- 课件、课堂 PPT：放入 `doc/ppt/`。
- 笔记、复习资料、知识点总结：放入 `doc/notes/`。
- 试卷、答案、考纲、期中期末相关材料：放入 `doc/exam/`。
- 老师布置的作业内容（题目、要求、指导书）：平时作业放入 `doc/tasks/homework/`，大作业、课程设计放入 `doc/tasks/project/`，实验作业放入 `doc/tasks/lab/`；学生提交的作业见 `assignment/`。
- 平时作业：按 `昵称/` 格式放入 `assignment/homework/`。
- 课程设计、大作业、完整系统、前后端工程：源代码不直接入仓；在 `assignment/project/` 下仅维护一个 `README.md`，链接到自己的外部仓库。
- 实验解答：按 `昵称/` 格式放入 `assignment/lab/`，存放实验报告；实验源码不直接入仓，应以链接形式维护。
- 图片、图表、截图：放入 `image/`。
- 压缩包：放入 `archive/`，除非压缩包本身是课程提交物或项目交付物的一部分，此时放入对应项目目录。

## 命名规则

- 课程下子目录与说明文件命名为 `学期编码-教师缩写`，如 `25262-wzh`，二者必须完全一致。
  - 学期编码：与学校学期编码一致（如 `25262`）；同课不同学期分别建目录。
  - 教师缩写：任课教师姓名的拼音首字母（小写），如 `wzh`、`dzj`。
- 其他分类目录名优先使用英文小写，如 `doc`、`assignment`、`image`、`archive`。
- 课程名、资料文件名可以保留中文，避免无意义重命名。
- 同类材料按课程已有语义命名；如需补充命名，可使用 `年份-类型-说明` 或 `实验编号-说明`。
- 学生作业子目录用昵称（如 `燕还故榻/`）；作业文件可带学号-姓名，如 `2352628-韦畅-作业1.pdf`。
- 不使用 `新建文件夹`、`资料`、`杂项` 等模糊目录名；确需临时存放时使用 `archive/` 并在说明文件中解释。

## 整理流程

1. 进入 `学期/课程名/学期-教师/` 目录，列出所有文件和子目录。
2. 读取已有 Markdown、txt 或可读说明文件，确认需要合并到 `<学期-教师>.md` 的内容。
3. 起草或更新 `<学期-教师>.md`，先保证说明入口完整。
4. 按文件类型和课程语义创建必要分类目录。
5. 移动资料文件，并同步更新 `<学期-教师>.md` 中的相对路径。
6. 检查是否存在空目录、失效链接、重复说明文件和明显误归类。
7. 使用 `git status` 查看改动范围，确认只改动当前课程相关文件。

## 注意事项

- 不要删除原始课程资料，除非它只是已合并的说明文件或明显重复文件。
- 已入仓但不符合「项目源代码不直接入仓」规范的源码，应迁移为外部链接后从本仓库删除；暂时没有链接时，在说明文件中以 `TODO` 标记。
- 不要修改二进制课程资料内容，只做移动和命名整理。
- 对 Git LFS 管理的大文件保持谨慎，移动后需要确认路径变化是否需要同步更新根 README 或 `.gitattributes`。
- 对包含构建产物的项目，优先记录现状；是否清理 `.o`、`.exe`、`node_modules` 等产物应单独决策。
- 每次只整理一门课程的一个学期-教师目录，避免大批量移动导致审查困难。
